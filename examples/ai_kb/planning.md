# 规划（Planning）

规划能力让 [[agent]] 能够将复杂任务分解为可执行的子步骤，并在执行过程中动态调整策略。
规划是区分简单 [[tool-use]] 调用与真正自主 Agent 的关键能力。

## 规划范式

### ReAct（Reason + Act）
最简单的规划框架：LLM 交替产生「思考」和「行动」，每步行动后观察结果再继续推理。
适合线性任务，但面对分支决策时容易陷入局部最优。

### Plan-and-Execute
先生成完整计划，再逐步执行。[[llm]] 在计划阶段可以全局权衡，执行阶段专注当前步骤。
[[claude]] 的长上下文窗口特别适合承载完整的计划与执行历史。

### Tree of Thoughts（思维树）
对每个决策点生成多个候选方案，通过搜索找到最优路径。计算成本高，适合需要探索的复杂问题。

## 规划与记忆

有效规划需要记忆支持：
- **短期记忆**：[[llm]] 上下文窗口，存储当前任务状态
- **长期记忆**：[[vector-db]] + [[knowledge-graph]]，存储历史经验与领域知识

[[rag]] 为规划提供知识支撑，[[semantic-search]] 让 Agent 在执行中动态检索所需信息。

## 评估

通过 [[evaluation]] 评估规划质量，关注任务成功率、步骤数效率和工具使用准确性。
[[guardrails]] 需要在规划层面加入约束，防止 Agent 生成危险的执行计划。
