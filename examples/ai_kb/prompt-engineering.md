# 提示词工程（Prompt Engineering）

Prompt Engineering 是通过精心设计输入文本，引导 [[llm]] 产出高质量输出的技术。
好的提示词设计能在不改变模型权重的前提下大幅提升任务性能，是 [[rag]] 与 [[agent]] 系统的核心。

## 核心技巧

### 角色设定（Role Prompting）
在系统提示中明确模型角色、背景和约束，帮助 [[claude]] 等模型建立任务上下文。

### 少样本示例（Few-shot）
在提示中提供 2–5 个输入输出示例，引导模型模仿期望格式。对结构化输出（JSON、Markdown）
效果显著，也是 [[knowledge-graph]] 实体抽取提示的常用手法。

### 思维链（Chain of Thought）
让模型先「思考再回答」，显著提升推理类任务准确性，是 [[agent]] [[planning]] 的基础。

### RAG 提示模板

```
你是一个专业助手。根据以下参考资料回答问题，不确定时说明。
参考资料：{context}
问题：{query}
```

好的 [[rag]] 提示明确限制模型只使用给定上下文，降低幻觉。

## 与微调的权衡

[[fine-tuning]] 把知识烧入权重，Prompt Engineering 通过上下文注入知识。
前者成本高但效果稳定，后者灵活但依赖上下文窗口。通过 [[evaluation]] 可以量化两者的效果差异。

## 护栏提示

系统提示也是 [[guardrails]] 的重要实现手段，定义模型的行为边界和拒绝策略。
